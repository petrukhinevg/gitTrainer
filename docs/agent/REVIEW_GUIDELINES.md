# Правила code review

- Findings first: баги, поведенческие риски, регрессии, нарушения контрактов и архитектуры, недостающие тесты.
- Naming, структура и читаемость попадают в findings только если создают реальный риск.
- Неблокирующие замечания можно выносить в `Нефункциональные наблюдения`.
- Если их нет, достаточно findings и кратких остаточных рисков.
