# Локальный старт агента

Используй этот файл как стартовую точку для каждого нового чата в этом репозитории.

## Кратко о продукте

- Продукт: одностраничное веб-приложение для изучения и практики Git через интерактивные сценарии, миссии и обратную связь по прогрессу.
- Цикл MVP: выбрать тренировочный сценарий -> изучить задачу и состояние репозитория -> ввести или собрать Git-действия -> получить проверку и объяснение.
- Backend отвечает за: определения сценариев, состояние упражнений, проверку ответов, отслеживание прогресса, оценивание, подсказки и правила генерации объяснений.
- Frontend отвечает за: маршрутизацию SPA, интерфейс тренировочного рабочего пространства, клиентское состояние взаимодействия, отображение прогресса и рендеринг обратной связи.

Проект ориентирован на пользователей, которые понимают базовые процессы разработки, но хотят безопасно и практически отрабатывать Git. Приложение должно помогать понять не только какую команду выполнить, но и почему именно она подходит для текущего состояния репозитория и цели задания.

## Кратко о репозитории

- Корневой проект: единый Gradle-проект со Spring Boot 4 и Java 21; сейчас ориентирован на backend и содержит документацию для будущего SPA-фронтенда.
- `frontend/`: запланированное SPA-приложение для тренировочного интерфейса; пока отсутствует, поэтому frontend-работа сейчас начинается с обновления roadmap и архитектуры.
- `docs/`: roadmap, архитектура, workflow и инструкции по работе с репозиторием.
- Другие важные каталоги: `src/main/java` для backend-кода приложения, `src/test/java` для автотестов, `src/main/resources` для конфигурации и статических ресурсов, когда они появятся.

## Маршрутизация сессии

Читай эти документы в каждом новом чате:

1. `docs/ROADMAP.md` для понимания границ продукта, рамок MVP и списка родительских задач.
2. `docs/ARCHITECTURE.md` для понимания границ backend/frontend и размещения пакетов.

Открывай эти документы только если они нужны для текущей задачи:

3. `AGENTS.md` для repo-wide ограничений, локальных доступов и карты инструкций.
4. `docs/TRACKER_WORKFLOW.md` для декомпозиции задач, рабочего workflow и гигиены смены scope.
5. `docs/agent/GIT_WORKFLOW.md` для веток, коммитов, push, PR, review-fix и проверок перед push.
6. `docs/agent/BOARD_WORKFLOW.md` для статусов issue, parent/sub-issue, labels и project board.
7. `docs/agent/GITHUB_AUTOMATION.md` для `gh`, GraphQL, linked branches и CLI-операций с GitHub.
8. `docs/BACKEND_ROADMAP.md` для backend-only задач или изменений API, данных и моделей.
9. `docs/FRONTEND_ROADMAP.md` для frontend-only задач или изменений UX/UI.
10. `.env` только если задаче действительно нужны локальные креды или токены.

## Маршрутизация по типу действия

- Если задача меняет рамки продукта, границы MVP или требует создания либо уточнения work items, смотри `docs/ROADMAP.md`.
- Если задача меняет зоны ответственности backend/frontend или размещение пакетов, смотри `docs/ARCHITECTURE.md`.
- Если задача связана с формулировкой scope, декомпозицией, разбиением на parent/sub-issue или сменой типа работы по ходу реализации, смотри `docs/TRACKER_WORKFLOW.md`.
- Если задача связана с ветками git, коммитами, push, созданием PR, исправлениями по ревью или проверками перед push, смотри `docs/agent/GIT_WORKFLOW.md`.
- Если задача связана со статусами issue, labels, структурой parent/sub-issue, статусом review или обновлением project board, смотри `docs/agent/BOARD_WORKFLOW.md`.
- Если задача связана с `gh`, GraphQL, linked branches или CLI-автоматизацией GitHub, смотри `docs/agent/GITHUB_AUTOMATION.md`.
- Если задача только про backend, смотри `docs/BACKEND_ROADMAP.md`.
- Если задача только про frontend, смотри `docs/FRONTEND_ROADMAP.md`.
- Если задаче нужны локальные креды, токены или аутентифицированные локальные инструменты, смотри `.env`.

## Заметки

- `AGENTS.md` теперь хранит только общие repo-wide правила и карту документов; профильные инструкции разнесены по отдельным файлам в `docs/agent/`.
- Локальные секреты храни в `.env`.
- Отдельного frontend-приложения в репозитории пока нет; не предполагай наличие `frontend/`, если оно не появится в одной из следующих задач.
